# 📌 Project Overview

Olive Store is a well-known Brazilian e-commerce platform, and this project focuses on analyzing its real-world business data to generate meaningful insights.

As an aspiring **Data Analyst** aiming to work in the **e-commerce industry**, I wanted to practice with a realistic dataset that reflects actual business operations. This project helped me understand how companies use data to improve sales, customer experience, and operational efficiency.

This project covers the complete data analytics workflow:

✔ Data Cleaning  
✔ Exploratory Data Analysis (EDA)  
✔ SQL Data Warehousing  
✔ Business Insights Generation  
✔ Interactive Power BI Dashboard  

---

# 🎯 Business Problem (STAR Method)

## ⭐ Situation
E-commerce businesses generate massive amounts of transactional data every day, including customer orders, payments, deliveries, product sales, and reviews.

However, raw data alone cannot help businesses make better decisions unless it is cleaned, analyzed, and transformed into actionable insights.

The Olive Store dataset provided an opportunity to analyze real-world e-commerce operations and understand customer behavior, sales trends, and delivery performance.

---

## ⭐ Task
The main objective of this project was to:

- Analyze customer purchasing behavior
- Identify top-performing products and categories
- Understand operational and delivery performance
- Generate business insights using SQL and Python
- Build an interactive dashboard for decision-making

## 📂 Dataset Information

Dataset Source:

🔗 [Brazilian E-Commerce Public Dataset by Olist](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce)

The dataset contains multiple CSV files representing different business entities:

- 👥 Customers
- 📦 Orders
- 🛍 Products
- 💳 Payments
- ⭐ Reviews
- 🚚 Sellers
- 🌍 Geolocation
- 🔄 Product Category Translation

These datasets together form a relational e-commerce database.

---

# 🧹 Data Cleaning & EDA

The analysis was performed using **Python in Jupyter Notebook**.

### 📚 Libraries Used
- Pandas
- NumPy
- Matplotlib
- Seaborn

During the cleaning process:
- Missing values were handled carefully
- Logical inconsistencies were validated
- Duplicate and irrelevant records were checked

Example:
Some canceled orders contained a purchase timestamp but had null delivery dates, which is logically correct because canceled orders are never delivered.

---

# 🗄 SQL Data Warehouse

After completing the EDA process, the cleaned datasets were connected to **SQL Server**.

A new database called:

```sql
OLIVE_DATAWAREHOUSE
```

was created for performing SQL analysis and generating deeper business insights.

### 📊 SQL Analysis Included:
- Revenue Analysis
- Top Selling Products
- Delivery Performance
- Customer Purchase Trends
- Seller Performance
- Payment Analysis
